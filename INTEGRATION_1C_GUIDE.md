# Подробное руководство по интеграции с 1С

## Как это работает

### Архитектура интеграции

```
┌─────────────────┐         HTTP/REST API         ┌──────────────────┐
│  1С (Клиент)    │ ────────────────────────────> │  FastAPI Server  │
│                 │ <──────────────────────────── │                  │
│  HTTPЗапрос()   │         JSON/XML ответ         │  /lessons/export │
│  ПрочитатьJSON()│                               │  /1c/json        │
└─────────────────┘                               └──────────────────┘
                                                           │
                                                           ▼
                                                   ┌──────────────────┐
                                                   │   MySQL Database │
                                                   │ additional_lessons│
                                                   │ lesson_enrollments│
                                                   └──────────────────┘
```

**Принцип работы:**
1. 1С делает HTTP-запрос к вашему API серверу
2. API сервер аутентифицирует запрос (проверяет Bearer token)
3. API извлекает данные из базы данных MySQL
4. API форматирует данные в JSON или XML
5. 1С получает данные и импортирует их в свои регистры/справочники

## Вариант 1: Периодический экспорт данных (рекомендуется)

### Шаг 1: Получение токена доступа

Для начала вам нужно получить токен доступа к API.

**В Flutter приложении (как админ):**
1. Войдите в систему как администратор
2. Токен сохраняется автоматически после входа

**Или через API напрямую:**
```http
POST /api/v1/token
Content-Type: application/x-www-form-urlencoded

username=admin&password=your_password&grant_type=password
```

**Ответ:**
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer"
}
```

### Шаг 2: Настройка в 1С

#### 2.1. Создайте обработку в 1С

Создайте внешнюю обработку с именем `ИнтеграцияЗанятий`.

#### 2.4. Импорт данных в регистры 1С

```bsl
// Процедура импорта занятий в справочник 1С
Процедура ИмпортироватьЗанятия() Экспорт
	
	Данные = ПолучитьЗанятия(ДатаНачала, ДатаОкончания);
	
	Если Данные = Неопределено Тогда
		Возврат;
	КонецЕсли;
	
	Занятия = Данные.lessons;
	
	Для Каждого Занятие Из Занятия Цикл
		
		// Находим или создаем элемент справочника "Занятия"
		Запись = Справочники.Занятия.НайтиПоКоду(Строка(Занятие.id), Истина);
		Если Не Запись.Заполнено() Тогда
			Запись = Справочники.Занятия.СоздатьЭлемент();
			Запись.Код = Строка(Занятие.id);
		КонецЕсли;
		
		Запись.Наименование = Занятие.title;
		Запись.Предмет = Занятие.subject;
		Запись.Преподаватель = НайтиПреподавателя(Занятие.tutor_id);
		Запись.Цена = Занятие.price;
		Запись.ДлительностьМинут = Занятие.duration_minutes;
		Запись.Онлайн = Занятие.is_online;
		Запись.МестоПроведения = Занятие.location;
		Запись.МаксСтудентов = Занятие.max_students;
		Запись.Записано = Занятие.enrolled_count;
		
		Запись.Записать();
		
	КонецЦикла;
	
	Сообщить("Импортировано занятий: " + Занятия.Количество());
	
КонецПроцедуры

// Процедура импорта записей в регистр сведений
Процедура ИмпортироватьЗаписиНаЗанятия() Экспорт
	
	Данные = ПолучитьЗаписиНаЗанятия(ДатаНачала, ДатаОкончания);
	
	Если Данные = Неопределено Тогда
		Возврат;
	КонецЕсли;
	
	Записи = Данные.enrollments;
	
	МенеджерЗаписи = РегистрыСведений.ЗаписиНаЗанятия.СоздатьМенеджерЗаписи();
	
	Для Каждого Запись Из Записи Цикл
		
		МенеджерЗаписи.Новая();
		МенеджерЗаписи.Занятие = НайтиЗанятие(Запись.lesson_id);
		МенеджерЗаписи.Студент = НайтиСтудента(Запись.student_id);
		МенеджерЗаписи.ДатаЗаписи = Дата(СтрЗаменить(Запись.enrollment_date, "T", " "));
		МенеджерЗаписи.Статус = Запись.status;
		МенеджерЗаписи.СтатусОплаты = Запись.payment_status;
		МенеджерЗаписи.Сумма = Запись.price;
		
		МенеджерЗаписи.Записать();
		
	КонецЦикла;
	
	Сообщить("Импортировано записей: " + Записи.Количество());
	
КонецПроцедуры
```

### Шаг 3: Настройка автоматической синхронизации

```bsl
// Процедура автоматической синхронизации
```

## Вариант 2: Файловый обмен (для изолированных систем)

Если 1С не имеет прямого доступа к интернету:

### Шаг 1: Экспорт данных в файл

Добавьте в ваше Flutter приложение кнопку экспорта или настройте серверный скрипт:

```bash
#!/bin/bash
# Скрипт для экспорта данных в файл

TOKEN="your_bearer_token_here"
API_URL="https://api.example.com/api/v1"

# Экспорт занятий
curl -H "Authorization: Bearer $TOKEN" \
     "$API_URL/lessons/export/1c/json?date_from=2024-01-01&date_to=2024-12-31" \
     > /shared/1c_exchange/lessons.json

# Экспорт записей
curl -H "Authorization: Bearer $TOKEN" \
     "$API_URL/lessons/enrollments/export/1c/json?date_from=2024-01-01&date_to=2024-12-31" \
     > /shared/1c_exchange/enrollments.json

# Экспорт финансов
curl -H "Authorization: Bearer $TOKEN" \
     "$API_URL/lessons/financial/export/1c/json?date_from=2024-01-01&date_to=2024-12-31" \
     > /shared/1c_exchange/financial.json
```

## Структура данных для импорта в 1С

### Рекомендуемая структура справочников и регистров в 1С:

#### Справочник "ЗанятияДополнительные"
- **Код** - ID из вашей системы
- **Наименование** - название занятия
- **Предмет** - строковый реквизит
- **Преподаватель** - ссылка на справочник "ФизическиеЛица"
- **Цена** - число (15, 2)
- **ДлительностьМинут** - число
- **Онлайн** - булево
- **МестоПроведения** - строка
- **МаксСтудентов** - число
- **Записано** - число

#### Регистр сведений "ЗаписиНаЗанятия"
- **Измерения:**
  - Занятие (ссылка на справочник "ЗанятияДополнительные")
  - Студент (ссылка на справочник "ФизическиеЛица")
- **Ресурсы:**
  - Сумма (число 15, 2)
- **Реквизиты:**
  - ДатаЗаписи (дата)
  - Статус (строка)
  - СтатусОплаты (строка)

#### Регистр накопления "ФинансыЗанятий"
- **Измерения:**
  - Занятие (ссылка на справочник)
  - Студент (ссылка на справочник)
- **Ресурсы:**
  - СуммаОплачено (число 15, 2)
  - СуммаОжидает (число 15, 2)

## Безопасность

### Рекомендации:

1. **Используйте HTTPS** - не передавайте токены по незащищенному соединению
2. **Ограничьте срок действия токена** - периодически обновляйте токены
3. **Используйте IP-фильтрацию** - ограничьте доступ к API только с IP адресов 1С сервера
4. **Логирование** - ведите журнал всех запросов от 1С
5. **Резервное копирование** - регулярно сохраняйте данные перед синхронизацией
