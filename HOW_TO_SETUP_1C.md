# Инструкция: Как настроить интеграцию с 1С в приложении

## Пошаговая инструкция

### Шаг 1: Откройте экран интеграций

1. Войдите в приложение как администратор школы
2. Перейдите в раздел **"Админ-панель"** (главный экран админа)
3. Найдите карточку **"Интеграции CRM"** и нажмите на неё
4. Нажмите кнопку **"Добавить интеграцию"** (внизу справа)

### Шаг 2: Заполните форму настройки

Вы увидите форму. Заполните её следующим образом:

#### 2.1. Тип CRM
- Выберите **"1С"** из выпадающего списка

#### 2.2. Название интеграции
- Введите название, например: **"1С:Бухгалтерия школы №123"** или **"1С:Управление образовательной организацией"**

#### 2.3. URL API вашей 1С
- Введите адрес API вашей 1С системы
- Пример: `https://1c-school.example.com/api/v1` или `http://192.168.1.100/base/ws/Integration`

**Где взять URL API:**
- Обратитесь к системному администратору 1С
- Или если веб-сервис уже настроен, найдите его адрес в настройках 1С
- Адрес обычно выглядит как: `http://ваш-сервер/base/ws/название_веб-сервиса`

#### 2.4. API ключ / Токен
- Введите токен доступа к API вашей 1С
- Это секретный ключ для авторизации

**Где взять API ключ/токен:**
- Обратитесь к системному администратору 1С
- Или создайте веб-сервис в 1С и получите токен доступа

**Как создать веб-сервис в 1С:**
1. В 1С откройте **"Администрирование"** → **"Интернет-поддержка"** → **"Публикация на веб-сервере"**
2. Выберите базу данных
3. Настройте публикацию (обычно на порт 80 или 443)
4. Получите URL публикации
5. Для токена: в 1С создайте пользователя для интеграции или используйте OAuth токен

#### 2.5. API Secret (опционально)
- Оставьте пустым, если в вашей 1С не требуется секретный ключ
- Или укажите, если это необходимо для вашего типа подключения

#### 2.6. Access Token (опционально)
- Оставьте пустым, если используете API ключ
- Или укажите Bearer токен, если ваша 1С использует OAuth авторизацию

### Шаг 3: Настройте параметры синхронизации

#### 3.1. Направление синхронизации
Выберите один из вариантов:
- **"Из приложения в CRM"** - данные передаются из приложения в 1С (рекомендуется для начала)
- **"Из CRM в приложение"** - данные передаются из 1С в приложение
- **"Двусторонняя"** - синхронизация работает в обе стороны

**Рекомендация:** Начните с **"Из приложения в CRM"** для экспорта данных о занятиях и записях в 1С.

#### 3.2. Частота синхронизации
Выберите, как часто синхронизировать данные:
- **"В реальном времени"** - данные передаются сразу при изменении (требует webhooks)
- **"Каждый час"** - синхронизация каждый час
- **"Ежедневно"** - один раз в день (рекомендуется для начала)
- **"Еженедельно"** - один раз в неделю
- **"Вручную"** - только когда вы нажимаете кнопку "Синхронизировать"

**Рекомендация:** Начните с **"Ежедневно"** или **"Вручную"** для тестирования.

### Шаг 4: Активируйте интеграцию

- Включите переключатель **"Активировать интеграцию"**
- Это включит автоматическую синхронизацию по расписанию

### Шаг 5: Сохраните интеграцию

- Нажмите кнопку **"Создать"** (или **"Сохранить"** если редактируете)

### Шаг 6: Тестируйте подключение

1. Вернитесь к списку интеграций
2. Найдите созданную интеграцию с 1С
3. Нажмите на неё, чтобы развернуть детали
4. Нажмите кнопку **"Тест"** для проверки подключения

**Если тест успешен:**
- Вы увидите сообщение "✅ Подключение успешно!"

**Если тест не прошел:**
- Проверьте правильность URL API
- Проверьте действительность токена/ключа
- Убедитесь, что 1С доступна из интернета
- Проверьте настройки безопасности в 1С (белый список IP адресов)

### Шаг 7: Запустите первую синхронизацию

1. После успешного теста нажмите кнопку **"Синхронизировать"**
2. Выберите тип данных для синхронизации:
   - **"lessons"** - дополнительные занятия
   - **"enrollments"** - записи на занятия
   - **"financial"** - финансовые данные
   - **"full"** - все данные сразу

3. Дождитесь завершения синхронизации
4. Проверьте данные в вашей 1С

## Что нужно подготовить в 1С перед настройкой

### Вариант 1: Веб-сервис 1С (рекомендуется)

1. **Настройте публикацию веб-сервиса:**
   - В 1С: **"Администрирование"** → **"Интернет-поддержка"** → **"Публикация на веб-сервере"**
   - Выберите базу данных
   - Настройте URL публикации
   - Запишите адрес (например: `http://ваш-сервер/base/ws`)

2. **Создайте веб-сервис** (если его нет):
   - В конфигураторе 1С создайте общий модуль с функциями для получения данных
   - Опубликуйте функции как веб-сервис
   - Укажите права доступа

3. **Получите токен доступа:**
   - Создайте отдельного пользователя в 1С для интеграции
   - Или используйте OAuth токен
   - Или настройте базовую HTTP-авторизацию

### Вариант 2: HTTP-сервисы 1С (для новых версий)

1. **Создайте HTTP-сервис:**
   - В конфигураторе 1С создайте HTTP-сервис
   - Настройте URL и обработчики запросов
   - Получите URL API

2. **Настройте авторизацию:**
   - Используйте API ключи или токены
   - Запишите токен для использования в приложении

## Проверка настроек

### Проверьте URL API:
Откройте в браузере:
```
http://ваш-сервер/base/ws
```
Должна открыться страница со списком веб-сервисов или документация API.

### Проверьте токен:
Используйте curl или Postman для проверки:
```bash
curl -H "Authorization: Bearer ВАШ_ТОКЕН" \
     http://ваш-сервер/base/ws/api/v1/test
```

## Что синхронизируется

После настройки интеграции в 1С будут передаваться следующие данные:

1. **Дополнительные занятия:**
   - Название занятия
   - Предмет
   - Преподаватель
   - Цена
   - Количество записанных студентов

2. **Записи на занятия:**
   - ФИО студента
   - Название занятия
   - Дата записи
   - Статус (записан, завершен, отменен)
   - Статус оплаты (ожидает, оплачено)
   - Сумма платежа

3. **Финансовые данные:**
   - Общий доход по занятиям
   - Оплаченные платежи
   - Ожидающие оплаты

## Решение проблем

### Ошибка "Connection failed"
- Проверьте URL API - должен быть правильным и доступным
- Проверьте, что 1С сервер запущен и доступен из сети
- Проверьте настройки файрвола

### Ошибка "Authentication failed"
- Проверьте правильность API ключа/токена
- Убедитесь, что токен не истек
- Проверьте права доступа пользователя в 1С

### Ошибка "Invalid URL"
- Убедитесь, что URL начинается с `http://` или `https://`
- Проверьте, что URL не содержит лишних пробелов

### Данные не синхронизируются
- Убедитесь, что интеграция активирована
- Проверьте расписание синхронизации
- Запустите синхронизацию вручную и проверьте логи

## Следующие шаги

После успешной настройки интеграции:

1. Проверьте первую синхронизацию вручную
2. Убедитесь, что данные корректно передаются в 1С
3. Настройте маппинг полей (если нужно)
4. Включите автоматическую синхронизацию

## Поддержка

Если возникли проблемы:
1. Проверьте раздел "История синхронизаций" в приложении
2. Проверьте логи 1С
3. Обратитесь к системному администратору 1С
4. Напишите в поддержку приложения с описанием проблемы
