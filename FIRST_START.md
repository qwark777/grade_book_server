# Первый запуск сервера

Это руководство поможет вам настроить сервер при первом запуске и создать учетную запись владельца.

## Шаг 1: Установка зависимостей

```bash
pip install -r requirements.txt
```

## Шаг 2: Настройка базы данных

1. Создайте базу данных MySQL:
```sql
CREATE DATABASE grade_book CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. Обновите настройки подключения в `app/core/config.py`:
```python
DB_HOST = "localhost"
DB_PORT = 3306
DB_USER = "your_username"
DB_PASSWORD = "your_password"
DB_NAME = "grade_book"
```

3. Инициализируйте таблицы (если они еще не созданы):
   - Таблицы создаются автоматически при первом запуске через `init_db()`
   - Или запустите миграции вручную

## Шаг 3: Создание первого владельца

**Это обязательный шаг!** Без владельца вы не сможете войти в систему.

Запустите скрипт создания владельца:

```bash
python scripts/create_owner.py --username owner --password your_secure_password
```

Или с указанием полного имени:

```bash
python scripts/create_owner.py --username owner --password your_secure_password --full-name "Иван Иванов"
```

### Параметры скрипта:

- `--username` (обязательно) - имя пользователя для входа
- `--password` (обязательно) - пароль (минимум 6 символов)
- `--full-name` (опционально) - полное имя владельца

### Примеры:

```bash
# Простой вариант
python scripts/create_owner.py --username admin --password Admin123!

# С полным именем
python scripts/create_owner.py --username owner --password SecurePass123 --full-name "Иван Петров"
```

### ⚠️ ВАЖНО:

1. **Сохраните учетные данные** в безопасном месте
2. Скрипт проверит, что владелец еще не создан (в системе может быть только один владелец)
3. Если пользователь с таким username уже существует, скрипт выдаст ошибку

## Шаг 4: Запуск сервера

```bash
python main_new.py
```

Или с uvicorn:

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload
```

## Шаг 5: Вход в приложение

1. Откройте мобильное приложение или веб-интерфейс
2. Введите созданные учетные данные:
   - **Username:** тот, который вы указали в `--username`
   - **Password:** тот, который вы указали в `--password`

## Что дальше?

После входа от имени владельца вы можете:

1. **Создать школы** через API или интерфейс
2. **Создать администраторов** для школ через `/api/v1/users/owners/create-admin`
3. **Настроить подписки** для школ
4. **Управлять всеми аспектами системы**

## Устранение проблем

### Ошибка "Владелец уже существует"
Если вы хотите создать нового владельца, сначала удалите существующего из базы данных:
```sql
DELETE FROM users WHERE role = 'owner';
```

### Ошибка подключения к базе данных
Проверьте:
- Запущен ли MySQL сервер
- Правильность настроек в `app/core/config.py`
- Существует ли база данных `grade_book`

### Забыли пароль владельца
Вы можете сбросить пароль напрямую в базе данных:
```sql
UPDATE users 
SET hashed_password = '$2b$12$...' 
WHERE role = 'owner' AND username = 'your_username';
```
(Используйте `get_password_hash()` из Python для генерации нового хеша)

## Безопасность

- Используйте **сильный пароль** (минимум 12 символов, буквы, цифры, символы)
- Не используйте простые пароли типа "admin", "password", "123456"
- Храните учетные данные в безопасном месте
- Регулярно меняйте пароль
