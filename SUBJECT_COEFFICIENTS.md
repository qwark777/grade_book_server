# Система коэффициентов предметов

## Обзор

Система коэффициентов предметов позволяет настраивать весовые коэффициенты для каждого предмета при расчете баллов. Это позволяет учитывать важность и сложность различных предметов при подсчете общей успеваемости.

## Принцип работы

### Базовые баллы
Каждая оценка имеет базовый балл согласно таблице:

| Оценка | Базовый балл | Описание |
|--------|--------------|----------|
| 5      | +10         | Отлично  |
| 4      | +5          | Хорошо   |
| 3      | -3          | Удовлетворительно |
| 2      | -6          | Неудовлетворительно |

### Расчет итоговых баллов
Итоговые баллы рассчитываются по формуле:
```
Итоговый балл = Базовый балл × Коэффициент предмета
```

### Примеры расчета

**Математика (коэффициент 1.5):**
- Оценка 5: 10 × 1.5 = 15 баллов
- Оценка 4: 5 × 1.5 = 7.5 баллов
- Оценка 3: -3 × 1.5 = -4.5 баллов
- Оценка 2: -6 × 1.5 = -9 баллов

**Физкультура (коэффициент 0.8):**
- Оценка 5: 10 × 0.8 = 8 баллов
- Оценка 4: 5 × 0.8 = 4 балла
- Оценка 3: -3 × 0.8 = -2.4 балла
- Оценка 2: -6 × 0.8 = -4.8 баллов

## Структура базы данных

### Таблица subjects
```sql
CREATE TABLE subjects (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) UNIQUE NOT NULL,
    coefficient DECIMAL(3,2) DEFAULT 1.00
);
```

### Коэффициенты по умолчанию
- **Математика**: 1.5 (важный предмет)
- **Физика**: 1.5 (важный предмет)
- **Английский язык**: 1.4 (важный предмет)
- **Информатика**: 1.3 (важный предмет)
- **Химия**: 1.3 (важный предмет)
- **Русский язык**: 1.2 (важный предмет)
- **Биология**: 1.1 (средний предмет)
- **История**: 1.0 (стандартный предмет)
- **География**: 1.0 (стандартный предмет)
- **Физкультура**: 0.8 (менее важный предмет)

## API Эндпоинты

### Получение предметов с коэффициентами
```http
GET /subjects-with-coefficients
```

**Ответ:**
```json
[
  {
    "name": "Математика",
    "coefficient": 1.5
  },
  {
    "name": "Физика", 
    "coefficient": 1.5
  }
]
```

### Обновление коэффициента предмета
```http
PUT /subjects/{subject_name}/coefficient
Content-Type: application/json

{
  "coefficient": 1.2
}
```

**Ответ:**

## Миграция

Для добавления поля `coefficient` в существующую базу данных запустите:

```bash
python migrations/add_subject_coefficient.py
```

Этот скрипт:
1. Добавляет поле `coefficient` в таблицу `subjects`
2. Устанавливает значение по умолчанию 1.00 для всех существующих предметов
3. Обновляет существующие записи

## Настройка коэффициентов

Коэффициенты можно настраивать через API или напрямую в базе данных:

```sql
-- Увеличить важность математики
UPDATE subjects SET coefficient = 2.0 WHERE name = 'Математика';

-- Уменьшить важность физкультуры
UPDATE subjects SET coefficient = 0.5 WHERE name = 'Физкультура';
```

## Рекомендации по настройке

- **Коэффициент 1.0**: Стандартные предметы
- **Коэффициент 1.2-1.5**: Важные предметы (математика, физика, языки)
- **Коэффициент 0.8-0.9**: Менее важные предметы (физкультура, ИЗО)
- **Коэффициент 0.5-0.7**: Дополнительные предметы

Система позволяет гибко настраивать весовые коэффициенты в зависимости от приоритетов школы и образовательной программы.
