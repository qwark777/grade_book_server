# Заполнение базы данных демо-данными для графиков

## Описание

Скрипт `seed_performance_data.py` заполняет базу данных демо-данными (оценками) для всех студентов за последние 60 дней. Это позволяет строить графики успеваемости даже если в системе еще нет реальных данных.

## Что делает скрипт

- Добавляет оценки для всех студентов за последние 60 дней
- Создает реалистичные тренды (улучшение/ухудшение успеваемости)
- Использует разные предметы из базы данных
- Генерирует оценки на основе реалистичных средних баллов (от 2.0 до 5.0)

## Запуск

### Вариант 1: Отдельно

```bash
cd grade_book_server
python3 seed_performance_data.py
```

### Вариант 2: Через init_performance_data.py (рекомендуется)

```bash
cd grade_book_server
python3 init_performance_data.py
```

Этот скрипт также создаст данные расписания и посещаемости.

## Результат

После запуска:
- ✅ Все студенты будут иметь оценки за последние 60 дней
- ✅ Графики в приложении будут строиться автоматически
- ✅ API эндпоинты вернут реальные данные из БД

## API эндпоинты

После заполнения данных будут работать:
- `GET /api/v1/performance-trends/{student_id}` - тренды успеваемости
- `GET /api/v1/grade-distribution/{student_id}` - распределение оценок
- `GET /api/v1/attendance-stats/{student_id}` - статистика посещаемости
- `GET /api/v1/subject-performance/{student_id}/{subject_id}` - данные по предмету

## Примечания

- Скрипт не удаляет существующие оценки (можно раскомментировать строку 75 в коде)
- Данные генерируются случайным образом, но с реалистичными трендами
- Каждый студент получает уникальный тренд (улучшение/ухудшение/стабильность)
