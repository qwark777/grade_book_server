# AI Advice - Быстрый старт

## Что уже сделано

✅ **Сервер (FastAPI)**:
- Endpoint `GET /api/v1/ai-advice/{student_id}` - собирает данные из БД и возвращает советы
- Логика анализа на основе правил (пока без нейронной сети)
- Правильный SQL-запрос с JOIN для получения названий предметов

✅ **Приложение (Flutter)**:
- Метод `getAiAdvice(userId)` в `ApiService`
- UI для отображения советов с приоритетами и категориями
- Автоматическая загрузка при открытии вкладки "Советы"

## Что нужно проверить

1. **Сервер запущен и работает**
   ```bash
   cd grade_book_server
   python main_new.py
   # или
   uvicorn app.main:app --host localhost --port 8001 --reload
   ```

2. **В базе данных есть оценки для студента**
   - Проверьте таблицу `grades` - должны быть записи для студента
   - Проверьте таблицу `subjects` - должны быть предметы
   - Проверьте связь через `subject_id`

3. **Пользователь авторизован**
   - Приложение должно быть залогинено
   - Токен должен быть валидным


3. Советы должны загрузиться автоматически

## Возможные проблемы

### 1. "Нет AI-советов"
- **Причина**: У студента нет оценок в базе или они не загружаются
- **Решение**: Проверьте, что в таблице `grades` есть записи для данного `student_id`

### 2. Ошибка 403 "Access denied"
- **Причина**: Пользователь пытается получить советы для другого студента
- **Решение**: Убедитесь, что `student_id` совпадает с ID текущего пользователя

### 3. Ошибка 500 при запросе
- **Причина**: Проблемы с SQL-запросом или структурой БД
- **Решение**: Проверьте логи сервера, убедитесь, что таблицы `grades`, `subjects` существуют

### 4. Советы не отображаются
- **Причина**: Ошибка в коде Flutter или неправильный формат ответа
- **Решение**: Проверьте логи Flutter, убедитесь, что API возвращает данные в правильном формате
